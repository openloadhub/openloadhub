# Support

## Community Support

Community support is best effort during the public alpha. Use public issues for reproducible bugs and focused feature requests, and follow the [Issue Triage](issue-triage.md) label and template guidance.

Good bug reports include:

- OpenLoadHub commit or release
- operating system and Docker version
- browser version
- exact steps to reproduce
- sanitized logs or screenshots
- whether the issue happens in the local demo or a modified deployment

## Security Reports

Do not report vulnerabilities in public issues. Use GitHub private vulnerability reporting or the private security advisory path documented in `SECURITY.md`.

## Commercial And Deployment Support

The open-source repository does not include guaranteed response times, production deployment ownership, custom plugin delivery, enterprise SSO implementation, or managed operations.

Those needs may be handled separately through commercial support or an enterprise offering.

## Out Of Scope For Community Support

- Debugging private infrastructure without a minimal public reproduction
- Reviewing secrets, private endpoints, or unsanitized logs
- Building custom enterprise integrations inside a public issue
- Production incident response for modified deployments
